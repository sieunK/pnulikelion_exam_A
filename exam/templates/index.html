RESTful 이란 무엇인가?

RESTful는 front-end를 신경쓰지않고 back-end 작업에만 집중하게 해줌으로써
back-end 작업을 효율적으로 할 수 있다는데 사실 잘 이해못했다.
그래도 이해한대로 서술해보자면, back-end에서 작업을 하고 이를 serializer를 통해 json 형식으로 front-end로 데이터를 보낼 수 있다는 거 같다.
최근에 RESTful의 필요성을 느낀 경험을 서술해보자면 php에서 exec를 통해 어떤 WEB API를 호출할 일이 있었는데
php를 exec로 실행시키려면 php, apache 등이 모두 root 권한을 줘야하만 실행이 가능했다.
그런데 이 WEB API가 RESTful api를 가지고 있다면 root 권한을 주지 않고도 실행시킬 수 있다고 했다.
RESTful이 굉장히 유용한 거 같긴한데 제대로 개발해본 적이 없어서 잘은 모르겠다.

serializer란 무엇인가?

POST 또는 GET 방식으로 통신을 할 때 파일을 json형식으로 변환해주는 편리한 기능을 제공한다.
예를 들어, ajax(front-end)와 php(back-end)와 같이 다른 언어끼리 통신을 할 때 주고받는 데이터의 형식을 맞춰줘야 이를 parsing해서
유용한 데이터로 활용할 수 있는데 주고받는 형식으로는 jquery, json, xml 등이 있다.
Django REST는 이중에서도 특히 json으로 파일 형식을 변환해준다.
